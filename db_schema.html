ScoreKeywords Database Schema Description

Overview

The ScoreKeywords database is designed to manage and analyze keywords within documents from various corpora. This schema supports the storage of document metadata, keyword scores, and reports on both documents and keywords. Below is a detailed description of each table and its intended use within the database.

Tables and Their Usage:

1. Corpus

  • Purpose: This table stores information about different document collections or corpora. Each corpus is uniquely identified and can contain multiple documents.
  • Columns:
    • CorpusId: Unique identifier for the corpus.
    • CorpusName: Name given to the corpus.
    • CorpusType: Type or category of the corpus.
    • CorpusPath: Physical or network path where the corpus is stored.
    • Username: The user responsible for this corpus.
    • IngestDate: When the documents in this corpus were ingested into the system.

2. Document

  • Purpose: This table holds details about individual documents within a corpus.
  • Columns:
    • DocId: Unique identifier for each document.
    • CorpusId: Foreign key linking to the Corpus table to show which corpus the document belongs to.
    • Name: The document's name or title.
    • Description: A brief description of the document's content.
    • CreatedDate: When the document was added to the database.

3. Chunk

  • Purpose: Documents are often too large to process in one go, so they are broken into smaller parts or chunks for analysis.
  • Columns:
    • ChunkId: Identifier for each chunk.
    • DocId: Foreign key to the Document table indicating which document this chunk belongs to.
    • Name: Name of the chunk, possibly for reference.
    • Description: Description or summary of the chunk.
    • CreatedDate: Date when the chunk was created.
    • ChunkSize: Size of the chunk in bytes, words, or another relevant unit.

4. Score

  • Purpose: This table records the score or relevance of keywords found within document chunks.
  • Columns:
    • ChunkId: Links to the Chunk table, showing which chunk this score pertains to.
    • KeywordId: Links to the Keyword table, showing which keyword is being scored.
    • Score: The numerical score representing the importance or frequency of the keyword.
    • ScoreDate: When the score was calculated.

5. Keyword

  • Purpose: Stores information about keywords that are being tracked or analyzed across documents.
  • Columns:
    • KeywordId: Unique identifier for each keyword.
    • Keyword: The actual keyword or phrase.
    • ShortDesc: A brief description or definition of the keyword.
    • LongText: Extended information or context about the keyword.

6. CorpusReport

  • Purpose: Contains reports generated about a corpus, possibly for summarization, analysis, or review purposes.
  • Columns:
    • ReportId: Identifier for the report.
    • CorpusId: Links to which corpus this report is about.
    • Title: Title of the report.
    • Type: Type of report (e.g., summary, detailed analysis).
    • Desc: Description of the report's content.
    • Summary: A summary of the findings or content.
    • Report: The full report text.
    • Reviewer: Who reviewed or generated the report.

7. DocumentReport

  • Purpose: Similar to CorpusReport but focused on individual documents, detailing findings, summaries, or reviews.
  • Columns: (Similar structure to CorpusReport but for individual documents.)

8. KeywordReport

  • Purpose: Provides detailed reports or studies on specific keywords, including their usage, impact, and relevance across documents.
  • Columns: (Structured similarly to CorpusReport and DocumentReport but focused on keywords.)

Usage:

  • Corpus Management: The Corpus table helps in organizing and managing collections of documents, which could be used for different projects or analyses.
  • Document Analysis: Documents are broken down into chunks for easier processing and scoring, which helps in detailed textual analysis or keyword extraction.
  • Keyword Scoring: The Score table is crucial for understanding how relevant or frequent keywords are within different parts of documents, aiding in SEO, content strategy, or thematic analysis.
  • Reporting: Reports on corpora, documents, and keywords provide insights into the data's content, trends, and relevance, helping stakeholders make informed decisions or understand content better.

This schema allows for a structured approach to managing and analyzing large sets of documents and keywords, providing a robust framework for content analysis, data mining, and information retrieval applications.
